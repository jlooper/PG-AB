PhoneGap, Cordova and AppBuilder

What's the difference?

Based on Cordova

PhoneGap is a distribution of Apache Cordova. Cordova powers PhoneGap, just like WebKit powers Chrome and Safari. AppBuilder is a platform that supports Cordova APIs and enables many different ways of building and distributing apps. Both PhoneGap and AppBuilder feature the integration of Cordova plugins like the accelerometer, camera, compass or the file system to access cross-platform native functionality. You also can use community-built Cordova custom plugins for added functionality such as bar code scanners and iBeacon readers. You can build great hybrid mobile apps using any of these tools. This site, built by members of the Developer Relations team at Telerik, is designed to help you differentiate between them.

AppBuilder

Telerik AppBuilder incorporates an extensive set of tools to build apps for iOS, Android and Windows Phone using a variety of strategies including the CLI, building in-browser, desktop apps, or as plugins for your preferred IDE.

Cordova

Apache Cordova is a set of device APIs for iOS, Android, Blackberry, Windows Phone, Palm WebOS, Bada, and Symbian that allows a mobile app developer to access native device functionality from JavaScript. Combined with a UI framework, Cordova allows you to build mobile apps with HTML, CSS, and JavaScript.

PhoneGap

PhoneGap is a solution for building cross-platform mobile apps using HTML, JavaScript, and CSS. It is an open-source distribution of Cordova that is managed by the Apache Foundation.

Workflow

How can I start developing a mobile app using PhoneGap vs. AppBuilder? What are my workflow options?

PhoneGap

After installing PhoneGap on your local computer, you can start building a new mobile app using a few CLI commands. Alternately, you can use the beta desktop client and drop your PhoneGap app into it to manage your app workflow. The app will then run on an IP address on your local machine for easy debugging.Many popular IDEs have PhoneGap plugins developed as well.

AppBuilder

AppBuilder offers an integrated solution to deliver mobile apps from concept to App store. You can develop apps using the CLI, the in-browser Platform, Sublime Text plugins, Visual Studio plugins or the Windows client. You can start your app work in the CLI, push to a Github repository, import that repository into the in-browser Platform, and continue work there.

Build

I need to build a version of my app. What's the difference between using PhoneGap Build and AppBuilder?

PhoneGap Build

PhoneGap developers might choose to use PhoneGap Build to build a testable and deployable version of their app. Using PhoneGap Build's web site, you upload a project's web assets in .zip format and the service compiles and packages an app. The developer then receives the download URLs for all platforms to test the app on a device. Alternately, the developer can configure the service to point to an opensource git repository and perform builds from that endpoint.

AppBuilder

Depending on how you prefer to develop your app, AppBuilder offers several different ways to build your app from code to deployable app. You can use the CLI commands to build a version of your app onto your local machine, or use the in-browser toolset to click 'build' to build a version of your app to install in the Companion App or directly to your device. Visual Studio users can build from that environment, as can Windows client users.

Sync

How do you refresh your app to test it on a device using PhoneGap vs. AppBuilder?

PhoneGap Syncing

When enabled in PhoneGap Build, "hydration" is available to PhoneGap app developers. Using Hydration, updates are pushed directly to the application installed on a device.Once a developer uploads a new build the end user of the application will be notified when restarting the application. Using the PhoneGap App, syncing occurs over wireless connections when you pair your computer to the installed app, which acts as a container for your app.

AppBuilder LiveSync

LiveSync enables you to see your code changes in real-time on connected mobile devices. When you deploy your application on connected physical or virtual devices, you can refresh your changes automatically by saving your changes. When you deploy your application in the AppBuilder Companion App on remote devices, you can refresh your changes by tapping and holding three fingers.

Test and Debug

Yikes, something's not working! How can you test and debug a mobile app using PhoneGap vs. AppBuilder?

PhoneGap Emulation

There are several ways to test an app's behavior. By typing 'phonegap emulate ios' in the command line you can see how your app looks on an iOS simulator, and similarly for Android emulators. PhoneGap Build offers a quick way to download builds for testing on a device, but you do need to include provisioning profiles for your iOS testers. The new PhoneGap app allows wireless emulation. Also available is an emulation service where you can enter a url and use Chrome's developer tools to debug. This emulation requires the Chrome browser and the Ripple emulation extension to be installed. Alternately, simply using Chrome's Dev Tools with the Device Emulation extension installed helps considerably to test your app. For on-device debugging you can make use of Safari's Developer Tools to debug remote code for iOS apps. For other devices, you can also use Weinre to get an idea of bugs that occur on a device. Finally, you can always use native development tools such as Xcode's console and Android's 'adb' toolset to debug.

AppBuilder

AppBuilder offers out-of-the-box a large selection of simulation options, including the ability to easily view many device sizes and types as well as Windows Phone. You can view an app's behavior using AppBuilder's built-in simulators or emulate the app using native simulators associated with Android, Windows Phone, and Xcode. In addition, when you're ready to test on device, AppBuilder offers the Companion App, a way for you to test your app on a device without provisioning, which eases iOS development in particular. Appbuilder includes several options for debugging yor app in a browser. You can use built-in developer tools to check javascript performance by inspecting the code in a browser on your local machine. Similar to PhoneGap, you can debug your app using Chrome or Safari developer tools or native software solutions. In addition, AppBuilder offers a suite of sophisticated testing tools. The new Test Portal allows you to write JavaScript test scripts and run them on devices in the cloud, greatly expanding the number of virtual devices on which to test apps.

Deploy

How can you deploy your app to the various App Stores using PhoneGap vs. AppBuilder?

PhoneGap

Once your app is complete, you can either build the app via native tools like Xcode and Android Studio or use PhoneGap build to deliver to you a properly-provisioned file. You will then deliver that file to the various app stores (iOS, Android, or Windows Phone) manually. Then you can complete the delivery of the app into production by adding the app assets into the various app stores.

AppBuilder

AppBuilder includes several helpful tools to ease the process of deploying your app to the various stores. From the CLI or the in-browser tools, you can build your properly-provisioned app and it will be automatically delivered to the app store of your choice by AppBuilder. AppBuilder also includes tools to help you create your splash screens and icons and handy methods to handle app provisioning from within the in-browser environment or the CLI. When you're ready, you can complete the app delivery process (adding screenshots, keywords, video, descriptions, and other assets) in the App store and submit the project for approval by Apple or into production on Android.
